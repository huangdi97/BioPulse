# ADR-0005：项目线严格分离

## 日期
2026-05-28

## 状态
已接受

## 背景
同一公司下有两个独立项目线：**问年**（衰老管理 HM，定位：衰老管理的商业基础设施）和**三重对抗智能体集群**（药物发现 DD，AI 制药）。两个项目在同一 VPS 上运行，代码库独立。

## 决策
两个项目**不混入同一文档、不混入同一会话、不混入同一代码仓库**。问年文档中不提三重对抗术语（如"药物发现"→ 用"药物构造 DD"），三重对抗文档中不提问年。

## 理由
- **认知隔离**：混淆会导致推演灾难（衰老管理 ≠ 制药，业务逻辑不可复用）
- **文档独立**：每个项目都有自己的设计文档全景，混入对方术语会误导读者
- **估值独立**：两个项目的商业价值和投入模型完全不同

## 被否决的方案
- **统一文档树**：按"健康科技"大类归档 → 会在文档底层引入不当关联
- **统一术语表**："药物发现"在两个项目中含义不同

## 后果
- 若衡的记忆（Memory）中两个项目的信息隔离存储
- 任何时候不提另一项目
- 回答问年问题时禁用三重对抗类比 —— 不是原则问题，是纪律问题
