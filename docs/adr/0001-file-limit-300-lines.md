# ADR-0001：源文件不超过 300 行

## 日期
2026-05-23

## 状态
已接受

## 背景
项目代码由 AI 编码助手（OpenCode）生成。AI 在处理超长文件时容易出现上下文漂移、编辑错位、漏改逻辑等问题。在 Batch 2 代码审计中发现仓库最大文件达 2,054 行（`repositories.py`），超过 1,000 行的文件有 28 个。

## 决策
所有项目源文件（.py，测试文件除外）**不超过 300 行**。超出 300 行的文件，由 OpenCode 自行压缩/拆分，不手动处理。

## 理由
- **保持原子性**：300 行 = 1 个文件 1 个职责，降低认知负荷
- **AI 友好**：短文件减少 AI 上下文漂移和编辑错位
- **可测试**：小粒度文件可独立 mock 和测试
- **不做特例**：路由文件、模型定义、工具层一视同仁

## 被否决的方案
- **手动拆分**：由若衡手动拆分 → 违反"零容忍手动编码"纪律，且若衡不直接写代码
- **设 500 行**：阈值过高，仍需第二次拆分
- **不设限制**：放任不管 → 继续产生千行级 god file

## 后果
- 拆分初期大约 22 个文件需要处理
- 测试文件中偶尔有超限情况（如 `test_knowledge_graph.py` 1,280 行），不做强制
- 拆分后引入断链风险 —— 见 ADR-0007
